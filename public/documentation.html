Documentation Page

To build the server-side component of a “movies” web application. The web
application will provide users with access to information about different
movies, directors, and genres. Users will be able to sign up, update their
personal information, and create a list of their favorite movies.

URL Endpoints

REQUEST URL HTTP METHOD URL PARAMETERS REQUEST BODY DATA FORMAT RESPONSE BODY DATA FORMAT
Return a list of all movies to the user /movies GET /movies None A JSON object holding data about all the movies
Return data about a single movie by title to the user /movies/[Title] GET /movies/:title None A JSON object holding data about a single movie (title, genre, director):
{
"Title" : "Interstellar",
"Genre": "Sci-Fi",
"Director" : "Christopher Nolan"
}
Return data about a genre to the user movies/genres/[Name] GET /movies/genres/:name None A JSON Object holding data about a certain genre(e.g. “Sci-Fi”).
Return data about a director to the user /movies/directors/[Name] GET /movies/directors/:name None A JSON Object holding data about a certain director.

Allow users to add a movie to their list of favorites /users/[Username]/movies/[MovieID] POST /users/:username/movies/:MovieID None A JSON object holding the data of the user, including the movieId that is added in FavoriteMovies.
Allow users to remove a movie from their list of favorites /users/[Username]/movies/[MovieID] DELETE /users/:username/movies/:MovieID None A JSON object holding the data of the user and showing one less MovieId in FavoriteMovies.
Allow existing users to deregister /users/[Username] DELETE /users/:username None A text message indicating that a user has been deleted.
